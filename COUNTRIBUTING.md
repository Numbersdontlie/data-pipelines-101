# Contributing to Data Engineering Playground

Thank you for your interest! Contributions are welcome.

## Ways to Contribute

- 📚 Add new SQL exercises with solutions
- 🔄 Add new Airflow DAG examples
- 🧪 Add data quality tests
- 📊 Add Superset dashboard exports
- 🐛 Fix bugs or improve documentation

## Guidelines

1. **Fork** the repo and create a feature branch
2. Follow existing code patterns and naming conventions
3. Add comments explaining *why*, not just *what*
4. Include exercises when adding new learning modules
5. Open a Pull Request with a clear description

## Code Style

- Python: formatted with `black`, linted with `ruff`
- SQL: UPPERCASE keywords, snake_case names, aligned columns

Open a GitHub Discussion!
